# Настройка Vite для работы с сервером

Для того чтобы приложение могло обращаться к серверу, необходимо настроить dev-сервер Vite. Суть в следующем: все API-запросы отправляются на dev-сервер, а тот проксирует их дальше -- на настоящий API-сервер.

Реализуется это через блок `proxy` в файле `vite.config.js`:

```javascript
export default {
  /* Ваша конфигурация */
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:3000',
        rewrite: (path) => path.replace(/^\/api/, ''),
      },
    },
  },
}
```

В этом примере все запросы, URL которых начинается с `/api`, перенаправляются на `http://localhost:3000`.

Документация API-сервера доступна по адресу [http://localhost:3000](http://localhost:3000) — json-server покажет список доступных эндпоинтов.

> **Внимание!** При работе с удалённым сервером убедитесь, что адрес `target` в конфигурации proxy соответствует адресу вашего API-сервера.

Пример конфигурации прокси для проекта TaskBoard:

```javascript
proxy: {
      "/api": {
        target: "https://api.example.com/",
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, ""),
      },
    },
```
