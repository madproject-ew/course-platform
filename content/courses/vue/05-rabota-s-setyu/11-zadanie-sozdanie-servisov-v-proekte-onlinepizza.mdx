# Задание: создание сервисов в проекте OnlinePizza

На этом этапе вы создадите сервисный слой для работы с API. Каждый сервис будет отвечать за взаимодействие с конкретным ресурсом json-server.

## Шаги выполнения

### 1. Создайте базовый HTTP-клиент

Создайте файл `src/services/api.js`. В нём настройте экземпляр axios с базовыми параметрами:

- `baseURL` — укажите `'/api'` (запросы будут проксироваться на json-server через настройки Vite);
- заголовок `Content-Type: application/json`;
- перехватчик ошибок (interceptor) — в случае ошибки ответа выводите её в консоль и пробрасывайте дальше через `Promise.reject`.

Экспортируйте настроенный экземпляр для использования в остальных сервисах.

### 2. Сервис для типов теста

Создайте файл `src/services/dough.service.js`. Реализуйте функцию `fetchDough()`, которая выполняет `GET /dough` и возвращает данные.

### 3. Сервис для размеров

Создайте файл `src/services/sizes.service.js`. Реализуйте функцию `fetchSizes()`, которая выполняет `GET /sizes`.

### 4. Сервис для соусов

Создайте файл `src/services/sauces.service.js`. Реализуйте функцию `fetchSauces()`, которая выполняет `GET /sauces`.

### 5. Сервис для ингредиентов

Создайте файл `src/services/ingredients.service.js`. Реализуйте функцию `fetchIngredients()`, которая выполняет `GET /ingredients`.

### 6. Сервис для дополнительных товаров

Создайте файл `src/services/extras.service.js`. Реализуйте функцию `fetchExtras()`, которая выполняет `GET /extras`.

### 7. Сервис для заказов

Создайте файл `src/services/orders.service.js`. Реализуйте две функции:

- `fetchOrders()` — выполняет `GET /orders` и возвращает массив заказов;
- `createOrder(data)` — выполняет `POST /orders` с телом запроса `data` и возвращает созданный заказ.

### 8. Сервис для адресов

Создайте файл `src/services/addresses.service.js`. Реализуйте три функции:

- `fetchAddresses()` — выполняет `GET /addresses`;
- `createAddress(data)` — выполняет `POST /addresses`;
- `updateAddress(id, data)` — выполняет `PUT /addresses/:id`.

### 9. Сервис аутентификации

Создайте файл `src/services/auth.service.js`. Реализуйте функцию `login(email, password)`, которая выполняет `GET /users` с параметрами фильтрации по email и password (json-server поддерживает фильтрацию через query-параметры: `?email=...&password=...`). Если пользователь найден — возвращает его данные. Если нет — выбрасывает ошибку.

## Проверка

Убедитесь, что json-server запущен на `localhost:3000`. Откройте консоль разработчика в браузере. Импортируйте любой из созданных сервисов и вызовите его функцию. Во вкладке Network проверьте, что запрос отправлен корректно и данные получены в ответе.
