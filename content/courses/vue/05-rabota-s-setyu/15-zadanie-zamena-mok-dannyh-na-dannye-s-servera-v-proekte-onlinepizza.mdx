# Задание: замена мок-данных на данные с сервера в проекте OnlinePizza

Пришло время избавиться от моковых данных и перевести приложение на работу с реальным API. После выполнения этого задания все данные будут загружаться с json-server, а операции — отправляться на сервер.

## Шаги выполнения

### 1. Обновите хранилище конструктора пиццы

Откройте `src/stores/pizza.js`. Замените заглушку `loadOptions()` на реальную реализацию: вызовите сервисы `fetchDough()`, `fetchSizes()`, `fetchSauces()`, `fetchIngredients()` и запишите полученные данные в соответствующие массивы состояния. Вызовите `loadOptions()` при монтировании компонента `HomeView.vue` (например, в хуке `onMounted`).

### 2. Обновите хранилище корзины

Откройте `src/stores/cart.js`. Список дополнительных товаров (`extras`) теперь должен загружаться с сервера через `fetchExtras()` из `extras.service.js`. Добавьте действие для загрузки доступных дополнительных товаров и вызовите его при монтировании `CartView.vue`.

### 3. Обновите хранилище заказов

Откройте `src/stores/orders.js`. Замените заглушку `loadOrders()` на вызов `fetchOrders()` из `orders.service.js`. Доработайте `addOrder()`: теперь при создании заказа отправляйте `POST`-запрос через `createOrder()` из сервиса, а затем обновляйте локальный массив `orders` полученным ответом.

### 4. Обновите ProfileView

Откройте `src/views/ProfileView.vue`. Адреса пользователя должны загружаться с сервера через `fetchAddresses()` из `addresses.service.js`. Добавление и редактирование адресов тоже должны отправляться на сервер через `createAddress()` и `updateAddress()`.

### 5. Удалите моковые данные

Удалите директорию `src/mocks/`. Убедитесь, что в проекте не осталось импортов из этой директории. Все данные теперь поступают с сервера.

### 6. Добавьте состояние загрузки

В каждом хранилище, которое загружает данные с сервера, добавьте флаг `isLoading` (по умолчанию `false`). Перед началом запроса устанавливайте его в `true`, после завершения — в `false`. В компонентах отображайте текст «Загрузка...» или индикатор загрузки, пока `isLoading` равен `true`.

### 7. Добавьте обработку ошибок

Оберните вызовы сервисов в `try/catch`. В случае ошибки сохраняйте сообщение об ошибке в состоянии хранилища (например, поле `error`). В компонентах отображайте пользователю понятное сообщение, если данные не удалось загрузить.

## Проверка

Выполните следующие проверки:

1. Остановите json-server — в приложении должно отображаться состояние загрузки или сообщение об ошибке.
2. Запустите json-server — данные конструктора (тесто, размеры, соусы, ингредиенты) загружаются с сервера и отображаются корректно.
3. Оформите заказ — он сохраняется на сервере. Перезагрузите страницу — заказ по-прежнему отображается в истории (загружен с сервера).
4. Убедитесь, что в проекте не осталось обращений к `src/mocks/`.
