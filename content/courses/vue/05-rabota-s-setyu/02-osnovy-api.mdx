# Основы API

API (Application Programming Interface) -- это интерфейс прикладного программирования, представляющий собой набор определений и протоколов для создания и интеграции программного обеспечения. Благодаря API ваше приложение может взаимодействовать с другими приложениями или сервисами, не зная деталей их внутреннего устройства. Это существенно упрощает разработку и экономит ресурсы.

Сервис, инициирующий API-запрос, называется **клиентом**, а тот, что формирует и отправляет ответ, -- **сервером**.

## Виды API-архитектуры

Существует несколько подходов к проектированию API.

### REST API

Большинство современных API строятся на основе архитектуры REST (Representational State Transfer). REST -- это набор рекомендаций, направленных на создание масштабируемых, легковесных и понятных API-интерфейсов.

Ключевые принципы REST:

*   Разделение клиента и сервера. Взаимодействие строится по модели «запрос -- ответ»: клиент отправляет запрос, сервер возвращает ответ. Сервер не может инициировать запрос к клиенту, а клиент не отвечает серверу.
*   Единый интерфейс. Все запросы и ответы передаются по протоколу HTTP в определённом формате, что гарантирует совместимость произвольного клиента с произвольным сервером. Ответы обычно форматируются в JSON.
*   Отсутствие состояния. Каждое обращение клиента к серверу самодостаточно: сервер не хранит контекст предыдущих запросов.
*   Кеширование. Ответы сервера должны содержать указания о том, допускается ли кеширование ресурса и на какой срок.

### SOAP API

SOAP (Simple Object Access Protocol) -- протокол передачи данных по сети, который также может использоваться для построения API. SOAP стандартизирован консорциумом W3C и использует XML для кодирования информации. Благодаря строгой спецификации формата сообщений, SOAP API считаются более безопасными по сравнению с REST, однако жёсткие правила делают их более громоздкими и сложными в реализации.

### WebSocket API

WebSocket API -- технология, позволяющая установить двусторонний интерактивный канал связи между браузером и сервером. После установки соединения обе стороны могут обмениваться сообщениями в режиме реального времени: сервер отправляет данные в формате JSON, а клиент их обрабатывает.

### RPC API

RPC (Remote Procedural Call) -- самая простая из рассмотренных архитектур. В отличие от REST и SOAP, акцент которых на передаче данных, RPC-интерфейсы вызывают удалённые процедуры -- фактически выполняют сценарии на сервере. RPC-вызовы могут использовать JSON (протокол JSON-RPC) или XML (протокол XML-RPC). Хотя XML обеспечивает чуть более строгую типизацию, в остальном оба варианта схожи. RPC -- относительно простой и лёгкий способ выполнения кода в удалённых сетях.
