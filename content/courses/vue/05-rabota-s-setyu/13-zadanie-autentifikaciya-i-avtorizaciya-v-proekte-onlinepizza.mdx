# Задание: аутентификация и авторизация в проекте OnlinePizza

На этом этапе вы реализуете полноценную аутентификацию: от запроса к серверу до навигационных ограничений и обновления интерфейса.

## Шаги выполнения

### 1. Обновите сервис аутентификации

Откройте `src/services/auth.service.js`. Доработайте функцию `login`: она должна отправлять запрос на json-server с фильтрацией по email и паролю (`GET /users?email=...&password=...`). Если массив в ответе пуст — пользователь не найден, нужно выбросить ошибку. Если пользователь найден — верните его объект. Также добавьте сохранение данных пользователя в `localStorage`, чтобы состояние сохранялось между перезагрузками страницы.

### 2. Обновите хранилище аутентификации

Откройте `src/stores/auth.js`. Замените заглушку действия `login`:

- вызывайте `auth.service.js` для выполнения запроса;
- при успешном ответе сохраняйте пользователя в состояние хранилища и в `localStorage`;
- устанавливайте `isAuthenticated` в `true`.

Доработайте действие `logout`:

- очищайте `user` и `isAuthenticated` в хранилище;
- удаляйте данные пользователя из `localStorage`.

Добавьте инициализацию при создании хранилища: при запуске приложения проверяйте `localStorage` — если там сохранён пользователь, восстанавливайте его в состояние хранилища.

### 3. Добавьте навигационный guard

Откройте файл маршрутизатора (например, `src/router/index.js`). Добавьте глобальный навигационный guard через `router.beforeEach`. Защищённые маршруты (`/profile`, `/orders`) должны перенаправлять на `/login`, если пользователь не аутентифицирован. Можно отметить защищённые маршруты через `meta: { requiresAuth: true }` в описании маршрутов.

### 4. Обновите LoginView

Откройте `src/views/LoginView.vue`. Добавьте валидацию формы:

- поле email — обязательное, должно содержать корректный формат email;
- поле пароля — обязательное, минимум 6 символов.

При отправке формы вызывайте `authStore.login(email, password)`. Если вход не удался — отобразите сообщение об ошибке под формой. При успешном входе — перенаправьте пользователя на главную страницу через `router.push('/')`.

### 5. Обновите AppHeader

Откройте `src/components/AppHeader.vue`. Реализуйте два состояния шапки:

- если пользователь не аутентифицирован — показывайте ссылку «Войти», ведущую на `/login`;
- если пользователь аутентифицирован — показывайте имя пользователя и кнопку «Выйти», которая вызывает `authStore.logout()`.

## Проверка

Проверьте сценарий:

1. В `db.json` json-server должен быть хотя бы один пользователь (например, `user@example.com` / `password123`).
2. Перейдите на `/profile` без авторизации — должно перенаправить на `/login`.
3. Введите неверные данные — отображается сообщение об ошибке.
4. Введите корректные данные — происходит вход, в шапке появляется имя пользователя, защищённые маршруты становятся доступны.
5. Перезагрузите страницу — пользователь остаётся авторизованным (данные восстановлены из `localStorage`).
6. Нажмите «Выйти» — состояние очищается, шапка снова показывает ссылку «Войти».
