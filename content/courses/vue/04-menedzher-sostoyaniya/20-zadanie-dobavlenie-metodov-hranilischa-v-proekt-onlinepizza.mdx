# Задание: добавление методов хранилища в проект OnlinePizza

Действия (actions) — это функции внутри хранилища, которые изменяют его состояние. На этом этапе вы реализуете все необходимые действия для четырёх хранилищ.

## Шаги выполнения

### 1. Действия хранилища конструктора пиццы

Откройте `src/stores/pizza.js` и добавьте следующие функции:

- **`setDough(dough)`** — устанавливает выбранное тесто.
- **`setSize(size)`** — устанавливает выбранный размер.
- **`setSauce(sauce)`** — устанавливает выбранный соус.
- **`setIngredientQuantity(ingredientId, quantity)`** — обновляет количество указанного ингредиента в объекте `ingredientQuantities`.
- **`resetConstructor()`** — сбрасывает все выбранные значения к начальным: тесто, размер и соус становятся `null` (или первыми элементами из доступных опций), количество всех ингредиентов обнуляется.
- **`loadOptions()`** — пока оставьте как заглушку (пустая функция). В модуле работы с сетью она будет загружать данные с сервера.

### 2. Действия хранилища корзины

Откройте `src/stores/cart.js` и добавьте:

- **`addPizza(pizzaConfig)`** — добавляет сконфигурированную пиццу в массив `pizzas`. Объект `pizzaConfig` должен содержать всю информацию о пицце: тесто, размер, соус, ингредиенты, цену и количество.
- **`removePizza(index)`** — удаляет пиццу по индексу из массива.
- **`updatePizzaQuantity(index, quantity)`** — обновляет количество пиццы по индексу. Если количество становится равным нулю, пицца удаляется из корзины.
- **`addExtra(extra)`** — добавляет дополнительный товар в массив `extras`.
- **`updateExtraQuantity(extraId, quantity)`** — обновляет количество дополнительного товара. Если количество равно нулю, товар удаляется.
- **`clearCart()`** — полностью очищает корзину: обнуляет массив пицц, массив дополнительных товаров и адрес доставки.

### 3. Действия хранилища аутентификации

Откройте `src/stores/auth.js` и добавьте:

- **`login(email, password)`** — пока реализуйте как заглушку: принимает email и пароль, устанавливает `user` в объект-заглушку и `isAuthenticated` в `true`. Полноценная реализация с обращением к серверу будет добавлена в модуле работы с сетью.
- **`logout()`** — сбрасывает `user` в `null` и `isAuthenticated` в `false`.

### 4. Действия хранилища заказов

Откройте `src/stores/orders.js` и добавьте:

- **`addOrder(order)`** — добавляет новый заказ в массив `orders`.
- **`loadOrders()`** — заглушка, которая позднее будет загружать заказы с сервера.

### 5. Не забудьте экспортировать всё необходимое

Убедитесь, что все функции, которые будут вызываться из компонентов, возвращаются из setup-функции хранилища.

## Проверка

Откройте Vue DevTools и проверьте работу действий:

- вызов `setDough`, `setSize`, `setSauce` обновляет соответствующие поля в хранилище конструктора;
- `setIngredientQuantity` корректно обновляет количество ингредиента;
- `addPizza` добавляет элемент в массив корзины;
- `clearCart` полностью очищает состояние корзины;
- `login` и `logout` корректно переключают флаг `isAuthenticated`.
