# Геттеры хранилища

Геттеры являются полным аналогом вычисляемых свойств (computed) для состояния хранилища. Они объявляются через свойство `getters` в `defineStore()` и получают состояние в качестве аргумента:

```javascript
export const useProfileStore = defineStore('profile', {
  state: () => ({
    name: 'Иван Петров',
  }),
  getters: {
    nameWithPrefix: state => `Имя пользователя: ${state.name}`,
  },
})
```

Результат одного геттера можно использовать внутри другого:

```javascript
export const useProfileStore = defineStore('profile', {
  state: () => ({
    name: 'Иван Петров',
  }),
  getters: {
    getNameWithPrefix: state => `Имя пользователя: ${state.name}`,
    getFullProfile: state => ({
      name: state.getNameWithPrefix,
      age: state.age
    })
  },
})
```

## Передача аргументов в геттер

Иногда геттеру требуются входные параметры. Для этого геттер должен возвращать функцию:

```javascript
export const useProfileStore = defineStore('profile', {
  state: () => ({
    name: 'Иван Петров',
  }),
  getters: {
    getNameWithPrefix: state => `Имя пользователя: ${state.name}`,
    getFullProfile: state => ({
      name: state.getNameWithPrefix,
      age: state.age
    }),
    getGreetings: state => {
      return greeting => `${greeting} ${state.name}`
    }
  },
})
```

```xml
<template>
  {{ store.getGreetings('Привет') }}
</template>

<script setup>
 import { useStore } from './stores/profile'

 const store = useStore()

</script>
```

Важно учитывать, что при таком подходе геттеры теряют кеширование и работают как обычные [функции.](https://pinia.vuejs.org/core-concepts/getters.html#passing-arguments-to-getters)

## Использование с Options API

В Options API геттеры доступны точно так же, как и обычные свойства хранилища:

```xml
<template>
  {{ store.getGreetings('Привет') }}
</template>

<script>
import { mapState } from 'pinia'
import { useStore } from './stores/profile'

export default {
  computed: {
    ...mapState(useStore, ['getNameWithPrefix']),
  },
}
</script>
```
