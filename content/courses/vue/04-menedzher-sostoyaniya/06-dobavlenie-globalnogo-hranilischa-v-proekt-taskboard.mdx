# Шаг 1. Создание файлов хранилищ

Перенесём глобальные объекты — задачи, колонки и фильтры — в отдельные хранилища Pinia. Это избавит нас от транзитной передачи данных через props и events.

В проекте уже подключён менеджер состояния Pinia, дополнительная настройка не нужна.

Создайте директорию `src/stores` со следующими файлами:

*   `index.js`;
*   `columns.js`;
*   `filters.js`;
*   `tasks.js`;
*   `users.js`.

# Шаг 2. Создание хранилища колонок

Инициализируем хранилище для колонок:

```javascript
import { defineStore } from 'pinia'

export const useColumnsStore = defineStore('columns', {
	state: () => ({}),
	getters: {},
	actions: {}
})

```

Добавим экспорт в `index.js`:

```javascript
export { useColumnsStore } from './columns'
```
