# Задание: создание состояний в хранилищах OnlinePizza

Теперь необходимо наполнить каждое хранилище начальным состоянием. Структура данных должна быть приближена к тому, что приложение будет получать от API — ориентируйтесь на структуру файла `db.json`, который использует json-server.

## Шаги выполнения

### 1. Состояние хранилища конструктора пиццы

Откройте `src/stores/pizza.js` и определите следующее состояние:

- `doughOptions` — массив доступных вариантов теста (изначально пустой, позже будет заполняться с сервера)
- `sizeOptions` — массив доступных размеров пиццы
- `sauceOptions` — массив доступных соусов
- `ingredients` — массив всех ингредиентов
- `selectedDough` — выбранное тесто (объект или `null`)
- `selectedSize` — выбранный размер (объект или `null`)
- `selectedSauce` — выбранный соус (объект или `null`)
- `ingredientQuantities` — объект вида `{ [id]: quantity }`, где ключ — идентификатор ингредиента, значение — его количество

### 2. Состояние хранилища корзины

Откройте `src/stores/cart.js` и определите:

- `pizzas` — массив сконфигурированных пицц. Каждая пицца — объект с полями: тесто, размер, соус, список ингредиентов с количествами, количество данной пиццы, рассчитанная цена
- `extras` — массив дополнительных товаров, каждый элемент содержит сам товар и его количество (например, `{ item, quantity }`)
- `deliveryAddress` — объект адреса доставки или `null`

### 3. Состояние хранилища аутентификации

Откройте `src/stores/auth.js` и определите:

- `user` — объект текущего пользователя или `null`
- `isAuthenticated` — булево значение, `false` по умолчанию

### 4. Состояние хранилища заказов

Откройте `src/stores/orders.js` и определите:

- `orders` — массив заказов (изначально пустой)

### 5. Соответствие структуре API

Обратите внимание на структуру данных в `db.json` вашего json-server. Поля объектов в хранилищах (тесто, размеры, соусы, ингредиенты) должны соответствовать тому, что возвращает сервер. Это упростит интеграцию с API в следующих модулях.

## Проверка

Запустите приложение и откройте Vue DevTools. Перейдите во вкладку Pinia и убедитесь, что:

- каждое хранилище отображает своё начальное состояние;
- массивы инициализированы как пустые массивы `[]`;
- объекты-заглушки имеют значение `null`;
- булевы флаги имеют корректные значения по умолчанию.
