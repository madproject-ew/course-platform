# Pinia и Vuex

Разработчики с опытом работы на Vue могут задаться вопросом: что такое Pinia и куда делся Vuex?

Pinia зародилась как эксперимент — попытка переосмыслить менеджер состояния Vue с использованием Composition API. За основу была взята идея того, как мог бы выглядеть Vuex следующего поколения. Со временем команда Vue пришла к выводу, что Pinia уже реализует большинство возможностей, запланированных для Vuex 5, и приняла решение сделать её официальным менеджером состояния фреймворка.

По сравнению с Vuex, Pinia предлагает более лаконичный API в стиле Composition API и надёжный вывод типов при работе с TypeScript.

Основные отличия Pinia от Vuex версии 4 и более ранних:

*   Отсутствуют `mutations` — они считались избыточной абстракцией.
*   Не требуется создавать собственные обёртки для поддержки TypeScript: всё типизировано из коробки, а API спроектирован для максимального использования вывода типов.
*   Модули динамические по умолчанию — нет необходимости вручную регистрировать их в рантайме.
