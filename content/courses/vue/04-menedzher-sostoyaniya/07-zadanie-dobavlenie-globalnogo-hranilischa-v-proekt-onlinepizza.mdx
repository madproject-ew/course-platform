# Задание: добавление глобального хранилища в проект OnlinePizza

На этом этапе вы создадите систему хранилищ Pinia для управления глобальным состоянием приложения OnlinePizza. Каждое хранилище будет отвечать за свою функциональную область.

## Шаги выполнения

### 1. Установите Pinia

Если Pinia ещё не установлена в проекте, выполните в терминале:

```
npm install pinia
```

### 2. Инициализируйте Pinia в приложении

Откройте файл `src/main.js`. Импортируйте `createPinia` из `pinia`, создайте экземпляр Pinia и подключите его к приложению через `app.use()`.

### 3. Создайте хранилище конструктора пиццы

Создайте файл `src/stores/pizza.js`. В нём определите хранилище с помощью `defineStore`. Это хранилище будет отвечать за состояние конструктора: выбранное тесто, размер, соус, количество ингредиентов.

Используйте Composition API стиль (`defineStore` с setup-функцией) — этот подход будет единым для всех хранилищ проекта.

### 4. Создайте хранилище корзины

Создайте файл `src/stores/cart.js`. Это хранилище будет управлять массивом сконфигурированных пицц в корзине и выбранными дополнительными товарами.

### 5. Создайте хранилище аутентификации

Создайте файл `src/stores/auth.js`. Хранилище будет хранить данные текущего пользователя и флаг аутентификации.

### 6. Создайте хранилище заказов

Создайте файл `src/stores/orders.js`. Хранилище будет управлять историей заказов пользователя.

### 7. Убедитесь в единообразии стиля

Все четыре хранилища должны использовать один и тот же стиль определения — Composition API (`defineStore` с setup-функцией). Внутри setup-функции состояние объявляется через `ref` / `reactive`, вычисляемые свойства через `computed`, а действия — как обычные функции.

## Проверка

Запустите приложение и откройте Vue DevTools. Во вкладке Pinia должны отображаться все четыре хранилища: `pizza`, `cart`, `auth`, `orders`. Каждое из них пока может быть пустым — главное, что они зарегистрированы и доступны.
