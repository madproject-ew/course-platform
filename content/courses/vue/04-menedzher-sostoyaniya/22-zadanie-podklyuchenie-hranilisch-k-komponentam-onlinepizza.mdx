# Задание: подключение хранилищ к компонентам OnlinePizza

Это масштабный этап рефакторинга. Вы замените локальное состояние компонентов на данные из хранилищ Pinia. После выполнения этого задания всё приложение будет работать через централизованное управление состоянием.

## Шаги выполнения

### 1. Подключение хранилища конструктора к HomeView

Откройте `src/views/HomeView.vue`. Замените локальное реактивное состояние (выбранное тесто, размер, соус, ингредиенты) на данные из `usePizzaStore()`. Компоненты-селекторы (`DoughSelector`, `SizeSelector`, `SauceSelector`, `IngredientsSelector`) теперь должны читать данные из хранилища, а при изменении пользовательского выбора — вызывать действия хранилища.

### 2. Подключение конструкторских компонентов

Компоненты конструктора могут получать данные из хранилища двумя способами: через пропсы от `HomeView` или напрямую через `usePizzaStore()`. Оба подхода допустимы. Получение через пропсы делает компоненты более переиспользуемыми, а прямое обращение к хранилищу — более простым. Выберите подход и придерживайтесь его.

### 3. Кнопка «В корзину»

Реализуйте логику кнопки добавления пиццы в корзину на странице конструктора. При нажатии:

- вызывается `cartStore.addPizza()` с текущей конфигурацией пиццы из `pizzaStore` (тесто, размер, соус, ингредиенты, цена);
- затем вызывается `pizzaStore.resetConstructor()` для сброса конструктора в исходное состояние.

Кнопка должна быть заблокирована, если `pizzaStore.isPizzaReady` возвращает `false`.

### 4. Обновление AppHeader

Откройте `src/components/AppHeader.vue`. Подключите хранилища корзины и аутентификации:

- отображайте количество товаров в корзине из `cartStore.itemsCount` (например, рядом с иконкой корзины);
- если пользователь аутентифицирован (`authStore.isAuthenticated`), показывайте имя пользователя из `authStore.user?.name`. В противном случае — ссылку на страницу входа.

### 5. Подключение CartView

Откройте `src/views/CartView.vue`. Отобразите все данные из `cartStore`:

- список пицц с возможностью изменения количества через компонент `AppCounter`, вызывающий `cartStore.updatePizzaQuantity()`;
- список дополнительных товаров с управлением количеством через `cartStore.updateExtraQuantity()`;
- итоговые суммы из геттеров: `totalPizzasPrice`, `totalExtrasPrice`, `totalPrice`;
- кнопку «Оформить заказ», которая создаёт заказ в `ordersStore.addOrder()` с данными корзины и вызывает `cartStore.clearCart()`.

### 6. Подключение LoginView

Откройте `src/views/LoginView.vue`. При отправке формы вызывайте `authStore.login(email, password)`. После успешного входа перенаправляйте пользователя на главную страницу с помощью `router.push('/')`.

### 7. Подключение ProfileView

Откройте `src/views/ProfileView.vue`. Отобразите данные текущего пользователя из `authStore.user`: имя, email и другую доступную информацию.

### 8. Подключение OrdersView

Откройте `src/views/OrdersView.vue`. Отобразите список заказов из `ordersStore.orders`. Для каждого заказа покажите состав, дату и итоговую сумму.

## Проверка

Пройдите весь пользовательский сценарий:

1. Откройте конструктор — выберите тесто, размер, соус, добавьте ингредиенты.
2. Нажмите «В корзину» — пицца появляется в корзине, конструктор сбрасывается.
3. Перейдите в корзину — измените количество, убедитесь, что итоги пересчитываются.
4. Оформите заказ — корзина очищается, заказ появляется в истории.
5. Войдите в систему — имя пользователя отображается в шапке, защищённые страницы доступны.
6. Выйдите из системы — шапка возвращается к ссылке «Войти».
