# Методы хранилища

Методы (actions) хранилища представляют собой аналог методов в компонентах. Они определяются в свойстве `actions` объекта `defineStore()` и служат для описания бизнес-логики. В частности, именно в actions обычно размещаются запросы к серверу:

```javascript
import { defineStore } from 'pinia'

export const useStore = defineStore('users', {
  state: () => {
    return {
      users: []
    }
  },
  actions: {
    async fetchUsers () {
      try {
        const response = await fetch('https://api.example.com/users')
        this.users = await response.json()
      } catch (error) {
        console.error('Возникла ошибка запроса:', error)
      }
    }
  }
})
```

## Использование с Options API

> **Примечание:** Хелпер `mapActions` используется только в Options API. При работе с Composition API (`<script setup>`) вызывайте методы хранилища напрямую.

При работе с Options API доступен метод `mapActions`:

```xml

<script>
import { mapActions } from 'pinia'
import { useStore } from './stores/users'

export default {
  methods: {
    ...mapActions(useStore, ['fetchUsers'])
  },
  created () {
    this.fetchUsers()
  }
}
</script>
```

## Отслеживание вызова методов хранилища

Метод `$onAction` позволяет наблюдать за вызовами actions и их результатами:

```javascript
const unsubscribe = store.$onAction(
  ({
    name, // имя метода
    store, // хранилище
    args, // переданные аргументы
    after, // хук, выполняемый после завершения работы метода
    onError, // хук, выполняемый, если произошла ошибка метода
  }) => {
    after((result) => {})
    onError((error) => {})
  }
)

// Отписаться от наблюдения
unsubscribe()
```
