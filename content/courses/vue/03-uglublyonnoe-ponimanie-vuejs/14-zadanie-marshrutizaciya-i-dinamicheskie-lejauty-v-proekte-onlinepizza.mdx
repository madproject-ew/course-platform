# Задание: маршрутизация и динамические лейауты в проекте OnlinePizza

В этом задании вы подключите маршрутизацию к проекту и настроите систему динамических лейаутов. Не все страницы приложения должны выглядеть одинаково: например, страница логина не содержит шапку с навигацией. Используйте знания о `vue-router` из предыдущих уроков.

## Шаг 1. Установите vue-router

Если вы не выбрали Vue Router при создании проекта, установите его:

```
npm install vue-router
```

## Шаг 2. Создайте файл маршрутов

Создайте файл `src/router/index.js`.

Определите в нём маршруты приложения:

- `/` — `HomeView` (конструктор пиццы)
- `/cart` — `CartView` (корзина)
- `/login` — `LoginView` (страница входа)
- `/profile` — `ProfileView` (профиль пользователя)
- `/orders` — `OrdersView` (история заказов)

Для страниц `CartView`, `LoginView`, `ProfileView` и `OrdersView` пока создайте файлы-заглушки в `src/views/` с простым шаблоном, содержащим заголовок `<h1>` с названием страницы. Подробное наполнение страниц будет в следующем задании.

Создайте экземпляр роутера с помощью `createRouter` и `createWebHistory`. Экспортируйте его из файла.

Подключите роутер в `src/main.js` — вызовите `app.use(router)` перед `app.mount`.

## Шаг 3. Создайте SimpleLayout

Создайте файл `src/layouts/SimpleLayout.vue`.

Этот лейаут предназначен для страниц без шапки (например, страница входа). В его шаблоне — только `<main>` с `<slot>` внутри, без `AppHeader`.

## Шаг 4. Настройте динамические лейауты

Идея: каждый маршрут в мета-данных указывает, какой лейаут использовать. По умолчанию — `AppLayout` (с шапкой). Для страницы логина — `SimpleLayout` (без шапки).

В файле маршрутов для маршрута `/login` добавьте мета-поле:

```
meta: { layout: 'SimpleLayout' }
```

Остальные маршруты могут не указывать `layout` — для них будет использоваться лейаут по умолчанию.

## Шаг 5. Обновите App.vue

Откройте `src/App.vue` и перепишите его:

- Удалите прямое подключение `AppLayout` и `HomeView`
- Импортируйте оба лейаута: `AppLayout` и `SimpleLayout`
- С помощью `useRoute` из `vue-router` получите текущий маршрут
- Создайте вычисляемое свойство `layoutComponent`, которое возвращает нужный компонент лейаута на основе `route.meta.layout`. Если `meta.layout` равно `'SimpleLayout'` — возвращайте `SimpleLayout`, иначе — `AppLayout`
- В шаблоне используйте динамический компонент `<component :is="layoutComponent">` и внутри него разместите `<router-view />`

Таким образом, при переходе на `/login` будет использоваться `SimpleLayout`, а на всех остальных страницах — `AppLayout` с шапкой.

## Шаг 6. Обновите навигацию в AppHeader

Откройте `src/layouts/AppHeader.vue`.

Замените обычные ссылки `<a href="#">` на компоненты `<router-link>`:

- "Главная" — `to="/"`
- "Корзина" — `to="/cart"`
- "Профиль" — `to="/profile"`

Кнопку "Войти" пока оставьте как есть — позже она будет переключать между страницей логина и выходом из аккаунта.

При желании можно стилизовать активную ссылку с помощью класса `router-link-active`, который `<router-link>` добавляет автоматически.

## Проверка

Запустите проект и проверьте:

- Навигация по ссылкам в шапке переключает страницы без перезагрузки
- На страницах `/`, `/cart`, `/profile`, `/orders` отображается шапка с навигацией
- На странице `/login` шапка отсутствует — используется `SimpleLayout`
- URL в адресной строке обновляется при переходах
- Кнопка "Назад" в браузере работает корректно
