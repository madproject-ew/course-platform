# Рендер-функции

Рендер-функции формируют виртуальный DOM, на основе которого Vue выполняет отрисовку и обновление реального DOM. Они работают «под капотом» шаблонов, преобразуя каждый элемент и текстовый фрагмент в узлы виртуального DOM. При обнаружении изменений в виртуальном дереве Vue обновляет соответствующие части реального DOM.

Для программного создания виртуальных узлов Vue предоставляет функцию `h`. Она принимает три параметра:

*   тип создаваемого элемента,
*   объект атрибутов и свойств,
*   дочерние элементы.

Рассмотрим создание элемента `<h1>`:

```javascript
const element = h('h1', { class: 'my-class', innerHTML: 'Привет!' })
```

Эта запись создаёт элемент `<h1 class="my-class">Привет!</h1>`. Добавить его на страницу можно так:

```javascript
import { h } from 'vue'

export default {
  setup(props) {
    return () => h('h1', { class: 'my-class', innerHTML: 'Привет!' })
  }
}
```

При использовании `<script setup>` добавляется шаблон с элементом `render`:

```xml
<template>
  <render/>
</template>

<script setup>
const render = () => h('h1', { class: 'my-class', innerHTML: 'Привет!' })
</script>
```

В следующем примере создадим рендер-функцию для такого HTML-шаблона:

```xml
<div>
  <div v-if="showYes">yes</div>
  <span v-else>no</span>
</div>
```

Соответствующая рендер-функция выглядит так:

```javascript
import { h, ref } from 'vue'

const showYes = ref(true)

export default {
  setup() {
    return () => h('div', [showYes.value ? h('div', 'yes') : h('span', 'no')])
  }
}
```

Таким образом, рендер-функции позволяют программно конструировать узлы DOM, что открывает гибкие возможности для динамического построения интерфейса.
