# Шаг 1. Создание компонента TaskView

В этой демонстрации мы реализуем отображение детальной информации об отдельной задаче. Пользователь сможет просмотреть подробности, кликнув по задаче на доске.

После подключения маршрутизатора URL задачи примет вид `{host}/{task id}`, например `localhost:5173/2`.

Создайте компонент `TaskView.vue` в директории `src/views`.

# Шаг 3. Реализация закрытия диалога задачи

Реализуем возможность закрытия окна с деталями задачи. Для этого достаточно выполнить навигацию на корневую страницу `/`.

Обработчик в блоке `script`:

```javascript
import {useRoute, useRouter} from 'vue-router'

const router = useRouter()
const route = useRoute()

const closeDialog = function () {
  router.push('/')
}
```

Кнопка закрытия в шаблоне:

```html
<!--Закрытие задачи-->
<button
  class="task-card__close"
  type="button"
  @click="closeDialog"
/>
```

Обработчик на корневом элементе компонента:

```html

<div
  class="task-card"
  tabindex="0"
  ref="dialog"
  @click.self="closeDialog"
  @keydown.esc="closeDialog"
>
```

Атрибуты `tabindex` и `ref` необходимы для программной фокусировки на диалоговом окне, чтобы клавиша Esc срабатывала сразу при открытии. Добавим код фокусировки в блок `script`:

```javascript
import {ref, onMounted} from 'vue'

const dialog = ref(null)

onMounted(() => {
  // Фокусируемся на диалоговом окне, чтобы сработала клавиша Esc без дополнительного клика на окне
  dialog.value.focus()
})
```

# Шаг 4. Добавление шапки задачи

Шапка задачи включает три элемента: наименование, кнопку редактирования и дату создания.

Компонент TaskView получает id задачи из URL. Вычислим задачу по id из массива:

```javascript
import {computed} from 'vue'

// Передадим все задачи в компонент
const props = defineProps({
  tasks: {
    type: Array,
    required: true
  }
})

const task = computed(() => {
  return props.tasks.find(task => task.id == route.params.id)
})
```

Здесь намеренно используется нестрогое сравнение `==`, чтобы избежать дополнительных проверок типов, поскольку id может приходить и как строка `"4"`, и как число `4`.

Разметка шапки в шаблоне:

```html
<!--Шапка задачи-->
<div class="task-card__block">
  <div class="task-card__row">
    <!--Наименование задачи-->
    <h1 class="task-card__name task-card__name--min">
      {{ task ? task.title : '' }}
    </h1>
    <!--Кнопка редактирования задачи-->
    <a
      class="task-card__edit"
      @click="router.push({
                name: 'TaskEdit',
                params: { id: $route.params.id }
              })"
    >
      Редактировать задачу
    </a>
  </div>
  <!--Дата создания задачи-->
  <p class="task-card__date">
    {{ task.dueDate }}
  </p>
</div>
```

# Шаг 5. Вынесение общего кода в composables

В наших проектах применяется Composition API, который позволяет выделять общую логику в отдельные функции --- composables.

В компоненте `TaskView` нам нужно отображать дату создания задачи в удобочитаемом формате. Можно написать соответствующую функцию прямо внутри компонента, однако она вскоре пригодится и в других местах.

Добавим вспомогательную функцию `getTimeAgo` в файл `src/common/helpers.js` для определения того, сколько времени прошло с момента создания задачи:

```javascript
import {
  DAY_IN_MILLISEC,
  TAG_SEPARATOR,
  MONTH_IN_SEC,
  YEAR_IN_SEC,
  DAY_IN_SEC,
  HOUR_IN_SEC,
  MINUTE_IN_SEC
} from './constants';

// ...

export const getTimeAgo = date => {
  // Проверяем, если дата приходит в корректном формате
  if (isNaN(Date.parse(date))) {
    return '... время не указано ...';
  }
  const seconds = Math.floor((new Date() - Date.parse(date)) / 1000);

  function getFinalString(number, pronounce) {
    return `${number} ${pronounce} назад`;
  }

  // Определяем правильное окончание
  function getPronounce(number, single, pluralTwoFour, pluralFive) {
    return number === 1
      ? single
      : number > 1 && number < 5
        ? pluralTwoFour
        : pluralFive;
  }

  // Проверяем, если задача создана более года назад
  let interval = seconds / YEAR_IN_SEC;
  if (interval > 1) {
    const number = Math.floor(interval);
    const pronounce = getPronounce(number, 'год', 'года', 'лет');
    return getFinalString(number, pronounce);
  }
  // Проверяем, если задача создана более месяца назад
  interval = seconds / MONTH_IN_SEC;
  if (interval > 1) {
    const number = Math.floor(interval);
    const pronounce = getPronounce(number, 'месяц', 'месяца', 'месяцев');
    return getFinalString(number, pronounce);
  }
  // Проверяем, если задача создана более дня назад
  interval = seconds / DAY_IN_SEC;
  if (interval > 1) {
    const number = Math.floor(interval);
    const pronounce = getPronounce(number, 'день', 'дня', 'дней');
    return getFinalString(number, pronounce);
  }
  // Проверяем, если задача создана более часа назад
  interval = seconds / HOUR_IN_SEC;
  if (interval > 1) {
    const number = Math.floor(interval);
    const pronounce = getPronounce(number, 'час', 'часа', 'часов');
    return getFinalString(number, pronounce);
  }
  // Проверяем, если задача создана более минуты назад
  interval = seconds / MINUTE_IN_SEC;
  if (interval > 1) {
    const number = Math.floor(interval);
    const pronounce = getPronounce(number, 'минуту', 'минуты', 'минут');
    return getFinalString(number, pronounce);
  }
  return 'сейчас';
};
```

Создайте файл `composables.js` в директории `src/common`. Сюда мы будем добавлять переиспользуемую логику для нескольких компонентов. Может показаться, что мы нарушаем принцип KISS, вынося код заранее, но в данном случае мы уже точно знаем, что эти функции скоро понадобятся.

```javascript
import {computed} from 'vue'
import {getTimeAgo} from './helpers'

export const useTaskCardDate = task => {
  return computed(() => {
    return `# ${task.id} создана ${getTimeAgo(task.dueDate)}`
  })
}
```

Обновим шаблон `TaskView.vue`, заменив сырую дату на отформатированную:

```html
<!--Дата создания задачи-->
<p class="task-card__date">
  {{ useTaskCardDate(task) }}
</p>
```

Не забудьте сделать импорт:

```html

<script setup>
  //...
  import {useTaskCardDate} from '../common/composables'
  //...
</script>
```

# Шаг 6. Добавление исполнителя задачи и срока выполнения

Добавим ещё одну вспомогательную функцию в `src/common/helpers.js`:

```javascript
export const getReadableDate = date => {
  if (isNaN(Date.parse(date))) return ''
  const newDate = new Date(date);
  const year = newDate.getFullYear();
  const month = newDate.getMonth();
  const day = newDate.getDate();
  return `${day}.${month + 1}.${year}`;
};
```

Эта функция принимает дату в произвольном формате и возвращает её в виде `dd.mm.yyyy`.

Импортируем вспомогательные функции в блоке `script` для обработки аватара и даты:

```javascript
import {getReadableDate, getImage} from '../common/helpers'

const dueDate = computed(() => {
  return getReadableDate(task.value.dueDate || '')
})
```

Разметка секции исполнителя и срока выполнения:


# Шаг 9. Создание маршрута для TaskView

Для корректного отображения компонента добавим его в конфигурацию маршрутов. Как уже отмечалось, маршрут для деталей задачи имеет вид `localhost:5173/${id задачи}`.

Обновите файл `src/router/routes.js`:

```javascript
export default [
  {
    path: '/',
    name: 'HomeView',
    component: () => import('../views/HomeView.vue'),
    meta: {layout: 'AppLayoutMain'},
    children: [
      {
        path: '/:id',
        name: 'TaskView',
        component: () => import('../views/TaskView.vue'),
        meta: {layout: 'AppLayoutMain'}
      }
    ]
  },
]
```

Обратите внимание, что `TaskView` добавлен как дочерний маршрут для `HomeView`. Чтобы маршрут отрисовывался, необходимо добавить `router-view` в компонент `HomeView.vue`:

```html
...
<section class="desk">
  <!--      Отображение дочерних маршрутов-->
  <router-view :tasks="props.tasks"/>
  <!--      Шапка доски-->
  ...
```

`router-view` размещён в самом начале компонента `HomeView`, перед секцией «Шапка доски», и принимает массив задач через входной параметр.

# Шаг 11. Обработка перехода на страницу деталей задачи

Для перехода к деталям задачи нужно:

*   добавить обработчик клика на каждую задачу;
*   при клике выполнить навигацию на соответствующий маршрут.

В файле `TaskCard.vue` найдите элемент `<div class="task">` и добавьте обработчик перехода:

```html

<div
  class="task"
  @click="router.push({ path: `/${task.id}` })"
>
```

Также подключите маршрутизатор в блоке `script`:

```javascript
import {useRouter} from 'vue-router'

const router = useRouter()
```
