# Пользовательские директивы

Наряду со стандартными директивами вроде `v-model` и `v-show`, Vue позволяет создавать собственные пользовательские директивы.

Директива используется для реализации повторяющегося поведения, привязанного к DOM-элементам.

> Компоненты являются основным механизмом переиспользования кода. Однако для низкоуровневых операций с DOM пользовательская директива --- более подходящий инструмент.

Рассмотрим пример: нужно реализовать автоматическое фокусирование на элементе при его монтировании в DOM (без использования атрибута `autofocus`):

```javascript
import { createApp } from 'vue'
import App from 'App.vue'

const app = createApp(App)

/*
 * Регистрируем новую директиву v-autofocus
 * Название директивы указывается без префикса v-*
 */
app.directive('autofocus', {
  mounted (el) {
    el.focus()
  }
})
```

Жизненный цикл директивы включает следующие хуки (все они опциональны):

*   `created` --- вызывается до привязки атрибутов или слушателей событий к элементу. Полезен, когда нужно зарегистрировать обработчики, которые должны сработать раньше обычных `v-on`.
*   `beforeMount` --- вызывается при первой привязке директивы к элементу, перед монтированием родительского компонента. Подходит для единоразовой инициализации.
*   `mounted` --- вызывается после монтирования родительского компонента.
*   `beforeUpdate` --- вызывается перед обновлением VNode компонента.
*   `updated` --- вызывается после обновления VNode компонента и всех его дочерних VNode.
*   `beforeUnmount` --- вызывается перед размонтированием родительского компонента.
*   `unmounted` --- вызывается однократно при отвязке директивы от элемента после размонтирования родительского компонента.

Хуки принимают следующие аргументы:

*   `el` --- DOM-элемент, к которому привязана директива;
*   `binding` --- объект с дополнительными параметрами;
*   `vnode` --- элемент виртуального DOM, используемый Vue.js для построения реального HTML DOM;
*   `prevVnode` --- предыдущий vnode, доступен только в хуках `updated` и `beforeUpdate`.

После регистрации директива используется в шаблонах:

```xml
<input v-autofocus />
```

## Динамические директивы

Директивы могут быть динамическими --- изменяться могут как значение, так и аргумент. Например, создадим директиву для установки размера текста:

```xml
<span v-font-size:small>Я маленький текст</span>
<span v-font-size:medium>Я средний текст</span>
<span v-font-size:large>Я большой текст</span>
```

Для доступа к аргументу (small, medium, large) обращаемся к объекту `binding`:

```javascript
app.directive('font-size', {
  mounted (el, binding) {
    const sizes = {
      small: '12px',
      medium: '18px',
      large: '24px'
    }

    el.style.fontSize = sizes[binding.arg] ?? '12px'
  }
})
```

Объект `binding` предоставляет следующие свойства:

*   `instance` --- экземпляр компонента, в котором используется директива.
*   `value` --- значение, переданное директиве. Для `v-my-directive="1 + 1"` значением будет `2`.
*   `oldValue` --- предыдущее значение, доступно только в `beforeUpdate` и `updated`.
*   `arg` --- аргумент директивы (если указан). Для `v-my-directive:foo` аргументом будет `foo`.
*   `modifiers` --- объект модификаторов (если указаны). Для `v-my-directive.foo.bar` --- `{ foo: true, bar: true }`.
