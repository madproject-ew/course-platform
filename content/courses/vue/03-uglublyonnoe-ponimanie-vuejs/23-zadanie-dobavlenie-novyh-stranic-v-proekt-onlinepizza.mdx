# Задание: добавление новых страниц в проект OnlinePizza

В предыдущем задании вы создали маршруты и файлы-заглушки для страниц. Теперь пришло время наполнить эти страницы содержимым. Используйте навыки работы с формами, списками и компонентами, полученные в предыдущих уроках.

## Шаг 1. Создайте страницу входа

Откройте (или создайте) файл `src/views/LoginView.vue`.

Реализуйте форму авторизации:

- Заголовок `<h1>` с текстом "Вход"
- Форму `<form>` с двумя полями:
  - Поле email — `<label>` + `<input type="email">`, связанное через `v-model` с реактивной переменной
  - Поле пароля — `<label>` + `<input type="password">`, связанное через `v-model`
- Кнопку "Войти" (`<button type="submit">`)
- На событие `submit` формы повесьте обработчик с модификатором `prevent` (чтобы предотвратить перезагрузку страницы). Пока обработчик может просто выводить введённые данные в `console.log` — реальную авторизацию мы подключим при работе с сетью

Стилизуйте форму: расположите её по центру страницы, задайте ширину (например, `max-width: 400px`), используйте общие классы `.input` и `.btn--primary` из `common.css`.

## Шаг 2. Создайте страницу корзины

Откройте (или создайте) файл `src/views/CartView.vue`.

Реализуйте структуру страницы корзины:

- Заголовок `<h1>` с текстом "Корзина"
- Секция для списка выбранных пицц. Пока данных нет — покажите текст "Корзина пуста. Перейдите в конструктор, чтобы добавить пиццу". Предусмотрите место для будущего списка (оберните заглушку в условие `v-if` / `v-else`)
- Секция "Дополнительные товары". Отрисуйте список дополнительных товаров (extras) из моковых данных: название, цена и `AppCounter` для выбора количества. Создайте файл `src/mocks/extras.json` с данными из `db.json`
- Блок итоговой суммы — отображение общей стоимости заказа. Пока это может быть сумма только дополнительных товаров
- Кнопка "Оформить заказ" — стилизованная с классом `.btn--primary`. Пока при клике выводите данные в `console.log`

## Шаг 3. Создайте страницу профиля

Откройте (или создайте) файл `src/views/ProfileView.vue`.

Реализуйте страницу профиля пользователя:

- Заголовок `<h1>` с текстом "Профиль"
- Форма с полями данных пользователя:
  - Имя — `<input type="text">` с `v-model`
  - Телефон — `<input type="tel">` с `v-model`
  - Email — `<input type="email">` с атрибутом `readonly` (email нельзя менять)
- Кнопка "Сохранить" для отправки формы (пока без реальной логики сохранения)
- Секция "Адреса доставки":
  - Заголовок `<h2>` "Адреса доставки"
  - Список адресов (пока пустой — текст "Нет сохранённых адресов")
  - Кнопка "Добавить адрес" — при нажатии показывает форму с полями: улица, дом, квартира, комментарий. Для переключения видимости формы используйте реактивную переменную и директиву `v-if`

## Шаг 4. Создайте страницу истории заказов

Откройте (или создайте) файл `src/views/OrdersView.vue`.

Реализуйте страницу истории заказов:

- Заголовок `<h1>` с текстом "История заказов"
- Условная отрисовка: если заказов нет — текст "Пока нет заказов"
- Предусмотрите разметку для будущего списка заказов (оберните заглушку в `v-if` / `v-else` по условию длины массива заказов). Каждый заказ в будущем будет содержать: номер заказа, список пицц, итоговую сумму, статус

Создайте реактивную переменную `orders` как пустой массив — данные будут загружаться с сервера в следующих модулях.

## Шаг 5. Стилизуйте страницы

Добавьте блок `<style scoped>` к каждой созданной странице. Обеспечьте единообразный вид:

- Используйте общие классы из `common.css` (`.card`, `.btn`, `.btn--primary`, `.input`)
- Используйте CSS-переменные из `variables.css` для цветов и отступов
- Формы центрируйте, задавайте им ограниченную ширину
- Секции визуально разделяйте отступами

## Проверка

Запустите проект и пройдите по всем маршрутам:

- `/login` — отображается форма входа с полями email и пароль, шапка отсутствует (SimpleLayout)
- `/cart` — отображается страница корзины с заглушкой для списка пицц, секцией доп. товаров и кнопкой оформления
- `/profile` — отображается форма профиля с полями данных и секцией адресов
- `/orders` — отображается текст "Пока нет заказов"
- Навигация между страницами работает корректно через ссылки в шапке
- Нет ошибок в консоли браузера
