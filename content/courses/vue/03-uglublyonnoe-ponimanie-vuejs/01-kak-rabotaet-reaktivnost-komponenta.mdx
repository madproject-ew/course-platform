# Как работает реактивность компонента

Одной из ключевых особенностей Vue.js является его система реактивности. Давайте разберёмся, каким образом она устроена и функционирует.

## Что такое реактивность

Под реактивностью понимается парадигма, при которой приложение автоматически адаптируется к изменениям данных декларативным образом. Допустим, у нас объявлены две переменные `a` и `b`, содержащие числовые значения, и третья переменная `c`, вычисляемая как сумма первых двух.

```javascript
let a, b, c

a = 10
b = 20

c = a + b
```

Если бы значение `c` пересчитывалось автоматически при каждом обновлении `a` или `b`, это было бы проявлением реактивности. Однако стандартный JavaScript так не работает, и для реализации подобного поведения необходимы дополнительные механизмы.

## Какой код выполняется сейчас внутри Vue.js

Для автоматического пересчёта суммы при изменении переменных первым делом необходимо обернуть вычисление в функцию.

```javascript
const updateSum = () => {
  c = a + b
}
```

Чтобы фреймворк получил доступ к этой функции, применяется специальная обёртка, называемая эффектом. Она инициирует отслеживание непосредственно перед вызовом оборачиваемой функции. Ниже смоделируем реализацию такого поведения, чтобы лучше понять внутренние механизмы Vue.js.

Для начала обернём функцию обновления суммы в **пока не существующую** функцию `createEffect`.

```javascript
createEffect(() => {
  c = a + b
})
```

Теперь напишем саму функцию `createEffect`.

```javascript
/* Здесь мы будем хранить работающие в данный момент эффекты */
const effects = []

/* Функция принимает аргументом другую функцию — это видно из предыдущего кода */
const createEffect = (func) => {
  const effect = () => {
    /* Перед запуском переданной функции кладём эффект в массив */
    effects.push(effect)

    func()

    /* После выполнения функции удаляем эффект из массива */
    effects.pop()
  }

  /* После создания эффекта запускаем его */
  effect()
}
```

Теперь при запуске эффект помещается в массив, и для проверки текущих активных эффектов достаточно обратиться к массиву `effects`.

В архитектуре Vue.js эффекты служат фундаментом для реактивности. На их основе построены как отрисовка компонентов, так и вычисляемые свойства (`computed`).

## Отслеживание изменений внутри Vue.js

Стоит сразу отметить, что отследить переприсваивание обычных переменных напрямую не получится.

```javascript
/* Начальное значение */
a = 3

/* Новое значение */
a = 4
```

JavaScript попросту не предоставляет такого механизма. Однако есть возможность перехватывать обращения к свойствам объектов, и это открывает широкие возможности.

Vue.js оборачивает данные в `Proxy` с обработчиками методов `get` и `set`. Подробный разбор `Proxy` выходит за рамки курса, но вы можете изучить его самостоятельно в [документации](https://developer.mozilla.org/ru/docs/Web/JavaScript/Reference/Global_Objects/Proxy).

Для наших целей важно знать, что `Proxy` --- это объект-обёртка над другим объектом или функцией, позволяющий перехватывать операции с ними. Вот короткий пример:

```javascript
const myObject = {
  value: 0
}

const proxyObject = new Proxy(myObject, {
  get(target, property) {
    console.log('Перехватываем здесь')
    return target[property]
  }
})

console.log(proxyObject.value)
/* В консоли будет выведено:
 *
 * > Перехватываем здесь
 * > 0
 */
```

Помимо простого вывода в консоль, внутри обработчика можно выполнять произвольную логику --- мы ограничены лишь возможностями языка. Можно даже не возвращать значение из метода `get`.

При работе с методами через `Proxy` возникает сложность с контекстом `this`. Для её решения применяется `Reflect`, API которого описан [в статье MDN](https://developer.mozilla.org/ru/docs/Web/JavaScript/Reference/Global_Objects/Reflect).

С использованием `Reflect` приведённый выше код трансформируется следующим образом:

```javascript
const myObject = {
  value: 0
}

const proxyObject = new Proxy(myObject, {
  get(target, property, receiver) {
    return Reflect.get(...arguments)
  }
})

console.log(proxyObject.value)
```

Перейдём к реализации реактивности на основе `Proxy` с применением **абстрактной** функции `track`. Эта функция в нашем примере не имеет конкретной реализации, но условимся, что она принимает объект (`target`) и свойство (`property`), которое нужно отслеживать.

```javascript
const myObject = {
  value: 0
}

const proxyObject = new Proxy(myObject, {
  get(target, property, receiver) {
    /* Вызываем нашу абстрактную функцию отслеживания */
    track(target, property)
    return Reflect.get(...arguments)
  }
})

console.log(proxyObject.value)
```

Внутри `track` происходит проверка текущего активного эффекта (мы разобрали это ранее) и его привязка к переданным `target` и `property`. Так свойство становится зависимостью соответствующего эффекта.

Помимо этого, нужно повторно запускать эффект при изменении свойства отслеживаемого объекта. Для этого в `Proxy` используется обработчик `set`, внутри которого вызывается функция `trigger`. Её задача --- отыскать эффекты, зависящие от изменившегося свойства, и перезапустить их.

```javascript
const myObject = {
  value: 0
}

const proxyObject = new Proxy(myObject, {
  get(target, property, receiver) {
    track(target, property)
    return Reflect.get(...arguments)
  },
  set(target, property, value, receiver) {
    /* Вызываем нашу абстрактную функцию запуска зависимых эффектов */
    trigger(target, property)
    return Reflect.set(...arguments)
  }
})

console.log(proxyObject.value)
```

Итак, проксирование (оборачивание объектов в `Proxy`) даёт Vue.js возможность встраивать логику перехвата и реагирования на изменения значений.

## Отрисовка при изменениях значений Vue.js

Подобно вычисляемому свойству `computed`, функция `render`, в которую компилируется шаблон Vue, не анализирует, каким именно образом используются зависимые переменные. Она лишь фиксирует, что определённые значения задействованы на каком-то этапе выполнения. При изменении любого из этих значений происходит повторный вызов функции для генерации нового виртуального DOM.
