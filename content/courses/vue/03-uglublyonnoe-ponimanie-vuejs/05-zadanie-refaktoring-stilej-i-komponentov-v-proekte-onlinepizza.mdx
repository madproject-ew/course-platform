# Задание: рефакторинг стилей и компонентов в проекте OnlinePizza

По мере роста проекта в коде накапливаются повторяющиеся стили и паттерны. В этом задании вы проведёте рефакторинг: вынесете общие стили в отдельный файл и улучшите переиспользуемость компонентов.

## Шаг 1. Найдите повторяющиеся стили

Просмотрите стили в ваших компонентах: `AppHeader.vue`, `HomeView.vue`, компоненты конструктора (`DoughSelector`, `SizeSelector`, `SauceSelector`, `IngredientsSelector`, `PizzaPreview`), а также `AppCounter.vue`.

Обратите внимание на повторяющиеся паттерны:

- Отступы (padding) у карточек и секций
- Стили карточек (рамка, скругление, тень)
- Стили кнопок (фон, цвет текста, скругление, состояние hover)
- Стили полей ввода (если есть)

## Шаг 2. Создайте файл общих стилей

Создайте файл `src/assets/styles/common.css`.

Определите в нём CSS-классы для типовых элементов интерфейса:

- `.card` — базовый стиль карточки: рамка, скруглённые углы (`border-radius`), внутренние отступы, при необходимости тень
- `.btn` — базовый стиль кнопки: убрать стандартное оформление, задать отступы, скругление, курсор `pointer`
- `.btn--primary` — кнопка с основным цветом фона (используйте CSS-переменную `--color-primary` из `variables.css`)
- `.btn--secondary` — кнопка с альтернативным оформлением (например, рамка без заливки)
- `.btn--disabled` — стиль для заблокированной кнопки (приглушённый цвет, курсор `not-allowed`)
- `.input` — стиль для текстовых полей: рамка, скругление, отступы, фокусное состояние

Используйте CSS-переменные из файла `variables.css`, который вы создали ранее.

## Шаг 3. Подключите общие стили

Импортируйте файл `common.css` в ваш основной файл стилей (например, `src/assets/main.css`) или в `src/App.vue`. Убедитесь, что общие классы доступны во всех компонентах приложения.

## Шаг 4. Замените дублирующиеся стили

Пройдите по компонентам и замените дублирующиеся стили на общие классы:

- В кнопках компонентов конструктора используйте класс `.btn` (и модификаторы `.btn--primary`, `.btn--secondary` по необходимости)
- В карточках ингредиентов и превью используйте класс `.card`
- В `AppCounter` кнопки "+" и "-" стилизуйте через `.btn`

При этом компонент-специфичные стили оставьте в `<style scoped>` — выносить нужно только то, что реально дублируется.

## Шаг 5. Улучшите AppCounter

Компонент `AppCounter` используется в списке ингредиентов, а в дальнейшем будет использоваться и в корзине — возможно, с другим визуальным размером.

Добавьте проп `size` (тип String, допустимые значения: `'small'` и `'normal'`, по умолчанию `'normal'`).

В зависимости от значения пропа `size` применяйте к корневому элементу компонента соответствующий CSS-класс: `counter--small` или `counter--normal`. В стилях задайте для `counter--small` уменьшенные размеры кнопок, шрифта и отступов.

## Проверка

Запустите проект и убедитесь, что:

- Общие стили подключены и работают
- Визуальное оформление приложения не изменилось (рефакторинг не должен менять внешний вид)
- Компоненты используют общие классы вместо дублирующихся стилей
- В scoped-стилях компонентов стало меньше повторяющегося кода
- `AppCounter` корректно отображается в обоих размерах (`small` и `normal`)
