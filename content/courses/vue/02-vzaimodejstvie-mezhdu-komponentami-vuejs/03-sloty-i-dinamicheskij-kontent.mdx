# Слоты и динамический контент

Слоты — мощный инструмент Vue для создания переиспользуемых компонентов, хотя освоить эту концепцию с первого раза бывает непросто. Разберём несколько примеров и посмотрим, как применять слоты на практике.

## Что такое слоты

Слоты — это механизм Vue, позволяющий компоновать компоненты иначе, нежели через строгие родительско-дочерние отношения. Благодаря слотам можно размещать контент в произвольных местах компонента и делать компоненты более универсальными. По сути, слот — это зарезервированная ячейка внутри компонента, куда будет помещено содержимое.

Начнём с простого примера:

```xml
<!-- RedContent.vue -->
<template>
  <div class="red">
    <slot />
  </div>
</template>

<style>
  .red {
    color: red;
  }
</style>
```

Компонент содержит обёртку `div` с классом `.red` и может использоваться для окрашивания любого текста в красный цвет:

```xml
<template>
  <RedContent>
    Этот текст будет красным
  </RedContent>
</template>

<script setup>
  import RedContent from './RedContent.vue'
</script>
```

Если указать содержимое непосредственно внутри тега `<slot>`, оно отобразится по умолчанию — когда в компонент не передано дочерних элементов:

```xml
<!-- RedContent.vue -->
<template>
  <div class="red">
    <slot>Красный текст по умолчанию</slot>
  </div>
</template>

<style>
  .red {
    color: red;
  }
</style>
```

```xml
<template>
  <!-- Внутрь компонента не передано дочерних элементов -->
  <RedContent />
</template>

<script setup>
  import RedContent from './RedContent.vue'
</script>
```

В этом случае будет выведен красный текст «Красный текст по умолчанию».

## Именованные слоты — named slots

Компонент может содержать несколько слотов, но тогда они должны быть именованными. Именованные слоты удобны, например, при создании разметки страницы:

```xml
<!-- AppLayout.vue -->
<div class="wrapper">
  <header>
    <slot name="header" />
  </header>
  <main>
    <slot />
  </main>
  <footer>
    <slot name="footer" />
  </footer>
</div>
```

Здесь два именованных слота и один слот по умолчанию. Для размещения контента в именованном слоте используется конструкция:

```xml
<template v-slot:[имя слота]>
  <!-- Контент -->
</template>
```

Или её сокращённая форма:

```xml
<template #[имя слота]>
  <!-- Контент -->
</template>
```

Пример использования `AppLayout`:

```xml
<AppLayout>
  <template v-slot:header>
    Это будет показано в хедере
    <nav>
      <a href="/">Главная</a> |
      <a href="/contact">Контакты</a> |
    </nav>
  </template>

  <p>Здесь будет слот по умолчанию</p>

  <template v-slot:footer>
    <p>Это будет показано в футере</p>
  </template>
</AppLayout>
```

## Слоты с ограниченной областью видимости — scoped slots

Слоты способны передавать данные или функции дочерним компонентам. Рассмотрим пример:

```xml
<!-- CarColor.vue -->
<template>
  <div>
    <slot>{{ car.color }}</slot>
  </div>
</template>

<script setup>
  const car = {
    // Данные о машине
  }
</script>
```

По умолчанию отображается цвет автомобиля. Но если мы захотим показать дополнительный цвет машины и переопределить слот по умолчанию:

```xml
<!-- Родительский компонент -->
<template>
  <CarColor>{{ car.additionalColor }}</CarColor>
</template>
```

...то код не сработает, так как переменная `car` недоступна в родительском компоненте. Для решения этой задачи применяются scoped slots.

Изменим компонент:

```xml
<!-- CarColor.vue -->
<template>
  <div>
    <slot v-bind:car="car">{{ car.color }}</slot>
  </div>
</template>

<script setup>
  const car = {
    // Данные о машине
  }
</script>
```

Теперь свойство `car` привязано к слоту через `v-bind`, и мы можем использовать входные параметры в родительском компоненте:

```xml
<!-- Родительский компонент -->
<template>
  <CarColor v-slot:default="slotParams">
    {{ slotParams.car.additionalColor }}
  </CarColor>
</template>
```

Сокращённый вариант записи:

```xml
<!-- CarColor.vue -->
<template>
  <div>
    <slot :car="car">{{ car.color }}</slot>
  </div>
</template>

<script setup>
  const car = {
    // Данные о машине
  }
</script>
```

```xml
<!-- Родительский компонент -->
<template>
  <CarColor #default="slotParams">
    {{ slotParams.car.additionalColor }}
  </CarColor>
</template>
```

## В чём польза слотов

Компоненты проектируются с расчётом на повторное использование. Нередко передавать большое количество входных параметров нецелесообразно, и в таких ситуациях слоты оказываются более подходящим решением. Как правило, слоты наиболее полезны при разработке компонентов высшего порядка — `HOC` (Higher-Order Components).
