# Виды и обработка событий

В процессе разработки приложений нередко возникает необходимость реагировать на действия пользователя: клик по кнопке, перемещение курсора, ввод текста и другие взаимодействия со страницей.

> Полный перечень браузерных событий доступен в [документации](https://developer.mozilla.org/ru/docs/Web/Events).

В контексте Vue событие служит механизмом коммуникации между дочерними и родительскими элементами.

## Прослушивание событий

Для подписки на событие применяется директива `v-on`, которой передаётся метод, вызываемый при наступлении события.

> Сокращённая запись `v-on` — символ `@`.

Допустим, при нажатии на кнопку нужно вывести информацию в консоль. Компонент Vue, реализующий такое поведение, может выглядеть следующим образом:

```xml
<template>
  <button @click="onClick">Click me</button>
</template>

<script setup>
  const onClick = (event) => {
    console.log(event)
  }
</script>
```

При обработке DOM-события метод-обработчик получает единственный аргумент — объект события, описываемый интерфейсом [Event](https://developer.mozilla.org/ru/docs/Web/API/Event). Именно это событие клика и будет выведено в консоль.

Теперь предположим, что помимо объекта события в обработчик `onClick` необходимо передать дополнительные параметры, например явно указанный тип.

```javascript
const onClick = (event, type) => {
  console.log(event, type)
}
```

В этом случае синтаксис вызова в директиве `@click` меняется — для передачи дополнительных аргументов используется следующая запись:

```xml
<button @click="onClick($event, 'click')">Click me</button>
```

Первым аргументом передаётся `$event` — специальная переменная, всегда содержащая данные о событии. Если никаких дополнительных аргументов нет, запись можно сократить, как в первом примере. Таким образом, `@click="onClick($event)"` эквивалентно `@click="onClick"`.

Для обработки события не обязательно создавать метод — логику можно описать прямо в шаблоне:

```xml
<button @click="counter = counter + 1"> + 1 </button>
```

Тем не менее в крупных приложениях или при реализации сложной логики такой подход лучше не использовать.

## Модификаторы событий

Нередко требуется изменить стандартное поведение элементов интерфейса. Представим форму с кнопкой `type="submit"` — по умолчанию при нажатии на неё форма отправляется.

Чтобы предотвратить это в чистом JavaScript, используют `event.preventDefault()`. Однако Vue предоставляет более удобный механизм — модификаторы событий. Модификатор добавляется к событию по шаблону `@{событие}.{модификатор}="{обработчик}"`.

Для отмены стандартного поведения применяется модификатор `prevent`:

```xml
<template>
  <form @submit.prevent="onSubmit">
    <button type="submit">Отправить</button>
  </form>
</template>

<script setup>
  const onSubmit = () => {
    console.log('Форма не будет отправлена')
  }
</script>
```

Помимо `prevent` доступны и другие модификаторы:

*   `stop` — останавливает фазу всплытия (bubbling) события в DOM.
*   `capture` — переключает обработку на фазу перехвата (capturing). Подробнее о фазах событий — в [статье про Event.eventPhase](https://developer.mozilla.org/ru/docs/Web/API/Event/eventPhase).
*   `self` — обработчик сработает только при клике непосредственно по элементу.
*   `once` — обработчик вызовется только один раз.
*   `passive` — улучшает производительность на мобильных устройствах, устанавливая `passive=true` в `addEventListener`. Подробности — в [статье на MDN](https://developer.mozilla.org/en-US/docs/Web/API/EventTarget/addEventListener). Этот модификатор несовместим с `prevent`.

Модификаторы можно комбинировать, например: `@click.stop.prevent="method"`. Также допустимо использование без обработчика: `@submit.prevent`.

## Обработка событий нажатия клавиш

Vue предоставляет модификаторы не только для общих событий, но и для клавиш клавиатуры и кнопок мыши.

Псевдонимы часто используемых клавиш:

*   `enter`;
*   `tab`;
*   `delete` — перехватывает Delete и Backspace;
*   `esc`;
*   `space`;
*   `up`;
*   `down`;
*   `left`;
*   `right`;

Модификаторы для системных клавиш:

*   `ctrl`,
*   `alt`,
*   `shift`,
*   `meta`.

> На клавиатурах Apple клавиша meta обозначена символом ⌘ (Cmd). На клавиатурах Windows — символом ⊞ (Win).

Модификатор `exact` позволяет задать точную комбинацию системных модификаторов, необходимую для срабатывания события:

```xml
<!-- Сработает, даже если нажаты Alt или Shift -->
<button @click.ctrl="onClick">A</button>

<!-- Сработает, когда нажат Ctrl и не нажаты другие клавиши -->
<button @click.ctrl.exact="onCtrlClick">A</button>

<!-- Сработает, когда не нажаты клавиши-модификаторы -->
<button @click.exact="onClick">A</button>
```

Для мыши предусмотрены аналогичные модификаторы:

*   `left` — левая кнопка мыши;
*   `right` — правая кнопка мыши;
*   `middle` — средняя кнопка мыши.
