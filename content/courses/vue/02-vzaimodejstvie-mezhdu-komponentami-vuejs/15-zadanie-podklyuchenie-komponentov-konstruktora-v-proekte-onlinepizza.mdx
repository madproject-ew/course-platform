# Задание: подключение компонентов конструктора в проекте OnlinePizza

В этом задании вы соедините все компоненты конструктора в единое целое на главной странице. Данные будут передаваться через пропсы и события, а вы увидите, как работает однонаправленный поток данных во Vue.

## Шаг 1. Импортируйте компоненты конструктора

Откройте `src/views/HomeView.vue`.

Удалите из шаблона ранее написанную разметку секций тесто/размер/соус/ингредиенты — теперь их заменят отдельные компоненты.

Импортируйте все компоненты из `src/modules/constructor/`:

- `DoughSelector`
- `SizeSelector`
- `SauceSelector`
- `IngredientsSelector`
- `PizzaPreview`

Также импортируйте моковые данные из `src/mocks/` (если они ещё не импортированы).

## Шаг 2. Настройте реактивное состояние

В блоке `<script setup>` создайте (или оставьте ранее созданные) реактивные переменные:

- `selectedDough` — `ref`, по умолчанию первый элемент из массива теста
- `selectedSize` — `ref`, по умолчанию первый элемент из массива размеров
- `selectedSauce` — `ref`, по умолчанию первый элемент из массива соусов
- `ingredientQuantities` — `ref` с объектом, где ключ — `id` ингредиента, значение — количество (изначально все равны 0)

## Шаг 3. Подключите селекторы в шаблоне

Разместите компоненты в шаблоне и свяжите их с данными через `v-model`:

- `DoughSelector` — передайте массив теста через проп `items`, свяжите выбор через `v-model` с `selectedDough`
- `SizeSelector` — передайте массив размеров через `items`, свяжите с `selectedSize`
- `SauceSelector` — передайте массив соусов через `items`, свяжите с `selectedSauce`
- `IngredientsSelector` — передайте массив ингредиентов через `items`, свяжите с `ingredientQuantities`

## Шаг 4. Подключите PizzaPreview

Разместите `PizzaPreview` в шаблоне (например, справа от секций выбора или под ними).

Передайте в него текущие выбранные значения через пропсы:

- `:dough` — `selectedDough`
- `:size` — `selectedSize`
- `:sauce` — `selectedSauce`
- `:ingredients` — вычисляемое свойство, которое формирует массив выбранных ингредиентов

Создайте вычисляемое свойство `selectedIngredients` (с помощью `computed`). Оно должно пройти по массиву всех ингредиентов, отфильтровать те, у которых количество больше 0 (по данным из `ingredientQuantities`), и для каждого вернуть объект с полями ингредиента плюс поле `quantity`.

## Шаг 5. Добавьте кнопку "В корзину"

Под конструктором (или в секции превью) добавьте кнопку "В корзину".

При клике на кнопку соберите объект текущей конфигурации пиццы:

- `dough` — выбранный тип теста
- `size` — выбранный размер
- `sauce` — выбранный соус
- `ingredients` — массив выбранных ингредиентов с количествами (из вычисляемого свойства `selectedIngredients`)
- `quantity` — 1 (количество таких пицц)

Пока сохраните этот объект в `console.log` — подключение к корзине мы реализуем, когда создадим хранилище.

После добавления в "корзину" сбросьте конструктор к начальным значениям: первый тип теста, первый размер, первый соус, все количества ингредиентов обнулены.

## Проверка

Запустите проект и убедитесь, что:

- Все секции конструктора отображаются на странице через отдельные компоненты
- Изменение выбора теста, размера или соуса обновляет превью
- Изменение количества ингредиентов через `AppCounter` отражается в превью
- Расчёт цены в превью корректно учитывает все выбранные параметры
- Кнопка "В корзину" собирает данные (проверьте через консоль) и сбрасывает конструктор
