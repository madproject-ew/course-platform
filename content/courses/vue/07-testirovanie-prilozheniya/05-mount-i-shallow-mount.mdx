# Mount и Shallow Mount

Прежде чем приступить к написанию тестов, разберёмся с несколькими ключевыми понятиями Vue Test Utils.

**Vue Test Utils** — это API, построенное на концепции обёрток (wrapper). Библиотека позволяет изолированно монтировать компоненты, подставлять моковые данные (свойства, инъекции, события) и проверять результаты рендеринга и сгенерированные события. Для монтирования компонентов используется объект wrapper, предоставляющий удобный API для взаимодействия с экземпляром Vue-компонента.

**Wrapper** — объект, содержащий примонтированный компонент или VNode вместе с набором методов для его тестирования.

## Глупый компонент

Глупым (presentational) называется компонент, который:

*   ничего не знает о своём родителе и дочерних компонентах;
*   получает данные исключительно через входные параметры (props);
*   не взаимодействует с глобальным хранилищем;
*   не выполняет навигацию между маршрутами;
*   не совершает API-вызовов.

## Умный компонент

Умный (container) компонент самостоятельно взаимодействует с API и глобальным хранилищем, передаёт входные параметры дочерним компонентам и обрабатывает пользовательские события от них.

Метод `mount` из Vue Test Utils создаёт Wrapper с полностью примонтированным и отрендеренным компонентом Vue, включая все дочерние компоненты.

Метод `shallowMount` (или `mount(Component, { shallow: true })`) также создаёт Wrapper, однако вместо реальных дочерних компонентов подставляются заглушки (stubs). Это позволяет тестировать компонент в полной изоляции от его потомков.
