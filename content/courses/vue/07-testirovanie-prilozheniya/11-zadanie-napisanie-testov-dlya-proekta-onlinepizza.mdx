# Задание: написание тестов для проекта OnlinePizza

На этом этапе вы напишете набор тестов для ключевых частей приложения: геттеров хранилища, действий хранилища и компонента.

## Шаги выполнения

### 1. Убедитесь, что Vitest настроен

Откройте `vite.config.js` и проверьте, что в нём присутствует секция `test` с базовыми настройками. Если её нет — добавьте конфигурацию для Vitest. Убедитесь, что в `package.json` есть скрипт `test:unit`, запускающий `vitest`.

### 2. Напишите тесты для геттера хранилища конструктора

Создайте файл `src/stores/__tests__/pizza.test.js`. Напишите один-два теста для геттера `pizzaPrice`:

- задайте состояние хранилища с известными значениями (конкретное тесто, размер, соус и набор ингредиентов с определённым количеством);
- проверьте, что `pizzaPrice` возвращает ожидаемое числовое значение;
- измените один из параметров (например, размер) и убедитесь, что цена пересчитывается корректно.

### 3. Напишите тесты для действий хранилища корзины

Создайте файл `src/stores/__tests__/cart.test.js`. Напишите один-два теста:

- проверьте, что `addPizza` добавляет элемент в массив `pizzas` — после вызова длина массива должна увеличиться на 1, а добавленный элемент должен соответствовать переданным данным;
- проверьте, что `clearCart` полностью очищает массив `pizzas` и массив `extras`.

### 4. Напишите тесты для компонента

Создайте файл `src/components/__tests__/AppCounter.test.js`. Напишите один-два теста для компонента `AppCounter`:

- проверьте, что компонент отображает переданное значение;
- проверьте, что при клике на кнопку «+» генерируется событие увеличения значения;
- проверьте, что при клике на кнопку «-» генерируется событие уменьшения, и значение не опускается ниже нуля.

### 5. Настройте Pinia для тестов

При тестировании хранилищ и компонентов, которые используют Pinia, необходимо создавать тестовый экземпляр Pinia. Используйте `createTestingPinia` из пакета `@pinia/testing` — он позволяет задавать начальное состояние хранилища и при необходимости подменять действия. Подробности описаны в документации Pinia: https://pinia.vuejs.org/cookbook/testing.html

## Проверка

Запустите тесты командой:

```
npm run test:unit
```

Все написанные тесты должны успешно пройти. Убедитесь, что в выводе нет ошибок и предупреждений, связанных с неправильной инициализацией Pinia или отсутствующими зависимостями.
