# Шаг 1. Подготовка к тестированию компонентов

В этой демонстрации мы покажем, как применять методику TDD при создании компонентов.

Для работы потребуется дополнительный пакет `@pinia/testing`, который предварительно должен быть установлен в проекте.

Тестирование будем демонстрировать на новом компоненте, который воспроизведёт функциональность `src/views/HomeView.vue`.

Создадим директорию `src/views/__tests__`, добавим в неё файл `TestComponent.spec.js` и напишем базовую настройку:

```javascript
import { mount } from '@vue/test-utils'
import '@/stores/__tests__/mockServices'
import { describe, beforeEach, it, expect, vi } from 'vitest'
import TestComponent from '../TestComponent.vue'
import { createTestingPinia } from '@pinia/testing'
import { useColumnsStore, useTasksStore, useUsersStore } from '@/stores'
import router from '@/router'

describe('TestComponent', () => {
	let wrapper, usersStore, tasksStore, columnsStore
	beforeEach(async () => {
		// Создаём экземпляр Pinia
		const pinia = createTestingPinia({
			createSpy: vi.fn,
			stubActions: false,
		})
		// Создаём экземпляр компонента
		wrapper = mount(TestComponent, {
			global: {
				plugins: [pinia, router] // добавляем Pinia и vue-router к компоненту
			}
		})
		// Создаём экземпляры хранилищ
		usersStore = useUsersStore()
		tasksStore = useTasksStore()
		columnsStore = useColumnsStore()
		// Делаем изначальные запросы для заполнения хранилищ
		await usersStore.fetchUsers()
		await tasksStore.fetchTasks()
		await columnsStore.fetchColumns()
	})
})
```

Добавим первый тест, подтверждающий, что компонент успешно монтируется:

```javascript
import { mount } from '@vue/test-utils'
import '@/stores/__tests__/mockServices'

import router from '@/router'

describe('TestComponent', () => {
// ...
  	it('should render', async () => {
		expect(wrapper.exists).toBeTruthy()
	})
})
```

Если запустить тесты сейчас, они упадут — компонента ещё нет. Создадим `TestComponent.vue` в директории `src/views`:

```html
<template>
    <main class="content">
        <section class="desk">

        </section>
    </main>
</template>

<script setup>

</script>

<style lang="scss" scoped>

</style>
```

Теперь тест проходит успешно.

Реализуем разметку фильтра в компоненте:

```html
<template>
  <main class="content">
    <section class="desk">
      <!--      Отображение дочерних маршрутов-->
      <router-view/>
      <!--      Шапка доски-->

        <h1
            class="desk__title"

        >
          Design Coffee Lab
        </h1>
        <!--        Добавили кнопку для добавления новой колонки-->
        <button
            class="desk__add"
            type="button"
            data-test="desk-add"
            @click="columnsStore.addColumn"
        >
          Добавить столбец
        </button>

        <div class="desk__filters">
          <div class="desk__user-filter">
<!--            Список пользователей-->

              <li

                  :key="user.id"
                  :title="user.name"
                  data-test="user-filter"
                  class="user-filter__item"

                      :src="getPublicImage(user.avatar)"
                      alt="Аватар юзера"
                      width="24"
                      height="24"

                </a>
              </li>
            </ul>
          </div>
        </div>
      </div>
    </section>

<script setup>
```

# Шаг 6. Тестируем отображение задач в первой колонке

В завершающем тесте убедимся, что дочерние компоненты отрисовываются корректно, и проверим, что в первой колонке отображаются нужные задачи:

```javascript
import { mount } from '@vue/test-utils'
import '@/stores/__tests__/mockServices'
import { describe, beforeEach, it, expect, vi } from 'vitest'
import TestComponent from '../TestComponent.vue'
import { createTestingPinia } from '@pinia/testing'
import { useColumnsStore, useTasksStore, useUsersStore } from '@/stores'
import router from '@/router'

describe('TestComponent', () => {
	// ...
	it('should have tasks in the first column', () => {
		const firstColumn = wrapper.find('[data-test="column-target-area"]')
		const taskTitles = firstColumn.findAll('[data-test="task-title"]')
		expect(taskTitles.length).toBe(5)
		expect(taskTitles[0].text()).toBe('Задача № 2')
		expect(taskTitles[1].text()).toBe('Задача № 6')
		expect(taskTitles[2].text()).toBe('Задача № 10')
		expect(taskTitles[3].text()).toBe('Задача № 14')
		expect(taskTitles[4].text()).toBe('Задача № 18')
	})
})
```

Для корректной работы теста нужно добавить атрибуты `data-test` в компоненты колонки и задачи.

В компоненте `@/modules/columns/components/DeskColumn.vue` замените:

```html
<div class="column__target-area">
```

На:

```html
    <div
        data-test="column-target-area"
        class="column__target-area"
    >
```

В компоненте `src/modules/tasks/components/TaskCard.vue` найдите элемент заголовка задачи:

```html
        <h5
            class="task__title"
            :class="{ 'task__title--first': !task.user }"
        >
          {{ task.title }}
        </h5>
```

И добавьте `data-test="task-title"`:

```html
        <h5
            class="task__title"
            :class="{ 'task__title--first': !task.user }"
            data-test="task-title"
        >
          {{ task.title }}
        </h5>
```

Теперь компонент полностью покрыт тестами, и мы можем быть уверены в его корректной работе.
